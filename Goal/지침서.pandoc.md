# 🌊 프로젝트 온보딩 가이드: The Tide --- 커스텀 RPG 서버 v2

> **"바닷물처럼 차오르고 빠지는 세계 속에서, 짧고 굵게 강해지는 컴팩트
> 하드코어 RPG"**

## 1. 프로젝트 매니페스토: "왜 이 서버를 만드는가?"

### 🔴 바닐라 야생의 한계 (The Problem)

1.  **목적성 상실과 지루함 (The Void):** 엔더 드래곤을 잡고 나면 무엇을
    해야 할지 모르는 공허함에 빠집니다. 성장의 목표가 모호해지고
    반복적인 광질만 남습니다.

2.  **불친절함과 정체된 템포 (Flat Pacing):** 성장의 속도가 계단식이거나
    극단적으로 느리며, 환경의 변화가 없어 긴장감이 유지되지 않습니다.

3.  **불합리한 하드코어 (The Rage Quit):** 난이도를 높이기 위해 하드코어
    모드를 켜면 단 한 번의 실수로 모든 노력이 물거품이 되어 게임을 아예
    접게 만듭니다.

4.  **반복적인 단순 노동 (The Boredom Grind):** 광산, 농장, 낚시 등
    자원을 얻는 과정이 단순 매크로 작업에 가까워 도파민이 차오르지
    않습니다.

### 🔵 'The Tide v2'의 해결책 (The Vision)

우리는 "혼자 즐기는 싱글 야생"을 극도로 다이나믹하고 풍성하게
재창조합니다. 멀티플레이어 서버의 복잡하고 무거운 시스템은 걷어내고,
오직 **1인 플레이어의 몰입감과 성장 피드백**에 초점을 맞춥니다.

- **살아 숨 쉬는 환경:** 조수(Tide) 시스템이 실시간으로 월드의 난이도와
  보상을 회전시키며, 플레이어에게 강제적인 휴식과 짜릿한 도전의 주기를
  제공합니다.

- **명확한 수직 성장:** 내 장비와 강함이 전투력(Power Tier / GS)이라는
  직관적인 숫자로 표현되어 다음 목표를 명확하게 제시합니다.

- **스마트한 편의성:** 인벤토리를 어지럽히는 잡템을 조개와 진주라는
  화폐로 환산하고, 가상 상점을 통해 야생에서도 언제든 필요한 테크를 올릴
  수 있게 합니다.

- **도전과 극복의 서사:** 단순히 몬스터의 스탯만 올리는 것이 아니라,
  고유한 접두사(Affix)와 플레이어에게 복수하러 찾아오는 '네메시스'를
  통해 나만의 서사를 만듭니다.

## 2. 핵심 디자인 철학 (Design Philosophy)

AI 개발 에이전트는 모든 모듈과 이벤트를 설계할 때 아래의 세 가지 핵심
질문과 원칙에 부합하는지 항상 검증해야 합니다.

### 💡 세 가지 질문

1.  **"플레이어는 지금 무엇을 원하는가?"** (동기 부여)

2.  **"긴장감과 도파민은 어디서 오는가?"** (재미의 근원)

3.  **"이 시스템이 다른 시스템과 어떻게 맞물리는가?"** (유기적 연결성)

## 3. 유료 플러그인 대체: 가볍고 강력한 "자체 플러그인" 구현 전략

대형 서버용 유료 플러그인(MMOItems, MythicMobs 등)은 무겁고 설정이
복잡합니다. 혼자 즐기는 서버인 만큼 **Paper API의 순정 기능**을
극대화하여 가볍고 버그 없는 전용 모듈을 구축합니다.

### 🛡️ 1. 아이템 외형 관리 (CustomModelData 활용)

- **기존 유료 플러그인의 한계:** 복잡한 리소스팩 주입 및 매핑으로 인한
  렉과 충돌.

- **우리의 전략:** 마인크래프트 기본 기능인 `CustomModelData` 정수
  코드를 활용합니다. 플러그인은 아이템 생성 시 특정 코드(예: `1001`)만
  심고, 바닐라 리소스팩을 통해 외형을 동적으로 렌더링합니다.

### 📊 2. PDC(PersistentDataContainer) 기반 데이터 태깅 (★매우 중요★)

- **개발 규칙:** **"데이터는 PDC에, Lore(설명란)는 오직 시각
  표현용으로만."**

- 장비의 공격력, 소켓 정보, 강화 단계 등 모든 RPG 데이터는 아이템 객체의
  내부 메타데이터 공간(PDC)에 바이너리로 영구 저장합니다. 아이템 Lore
  텍스트를 파싱하여 값을 읽는 초보적인 실수를 절대로 범하지 마십시오.

### ⚔️ 3. 직관적인 전투 리스너 (Combat Event Listener)

- 복잡한 RPG 공식 대신, 플레이어가 몹을 타격할
  때(`EntityDamageByEntityEvent`) 장비의 PDC 데이터를 검사하여 작동하는
  직관적인 조건문 기반 시스템을 구축합니다.

  - 예: 무기에 `lifesteal: I` 태그가 있다면 대미지의 8%만큼 플레이어
    체력 회복(`EntityRegainHealthEvent`).

  - 예: 무기에 `lightning: II` 태그가 있다면 10% 확률로 피격 대상 위치에
    번개 소환.

### 🏰 4. 인스턴스식 전용 파밍 구역 (Deep Mine)

- 월드 전체를 복사하는 무거운 방식 대신, 특정 지하 영역을 격리된 방처럼
  지정하거나 주기적으로 리셋하는 미니 인스턴스 메커니즘을 사용합니다.

- 광산 내에서 사망할 경우 사망 위치에 아이템이 보존되는 대신, 강제로
  추방당하고 획득했던 광산 전용 전리품을 일부 소실하게 하여 "조금만 더
  깊이 갈까, 아니면 안전하게 복귀할까?"의 긴장감을 제공합니다.

## 4. AI 개발 에이전트를 위한 지침서 (Agent Guidelines)

이 프로젝트를 코딩할 때 AI는 반드시 다음 기술적 가이드라인을 엄격히
준수해야 합니다.

### 1️⃣ 기술 스택 및 구동 환경

- **서버 플랫폼:** Paper (최신 LTS 버전 호환)

- **언어:** Java 17 이상 / Bukkit 및 Paper API 활용

- **데이터 보존:** 플레이어의 화폐, 평판, 퀘스트 진행 상황은 YAML 기반
  로컬 데이터 저장 또는 경량 데이터베이스(SQLite)를 사용하며,
  비동기(Async) 처리로 서버 렉을 원천 봉쇄합니다.

### 2️⃣ 성능 최적화 원칙

- **이벤트 중심 설계 (Event-Driven):** `Scheduler`를 통해 모든 엔티티를
  주기적으로 스캔(`runTaskTimer`)하는 무거운 루프를 전면 배제합니다.

- **스폰 시점에 처리:** 접두사(Affix) 정예 몹 생성은
  `CreatureSpawnEvent`가 발생할 때 한 번만 연산하여 태그와 스탯을
  부여합니다.

### 3️⃣ 소프트 하드코어 죽음 설계

- 사망 시 플레이어 캐릭터를 삭제하는 대신, **"조류에 휩쓸림(Swept by
  Tide)"** 시스템을 구현합니다.

  - 사망 시 소지한 조개(화폐)의 10%가 유실됩니다.

  - 사망한 위치에는 커스텀 아머 스탠드 또는 상자 형태의 '유실물 비석'이
    세워집니다.

  - 플레이어는 부활 후 일정 시간 동안 공격력/방어력이 감소하는 약화
    디버프를 받으며, 시간 내에 비석으로 돌아가 아이템을 회수해야 합니다.

## 5. 결론 및 개발 방향성

"The Tide v2"는 단순히 몹이 세지고 아이템이 많이 떨어지는 서버가
아닙니다.

**바닷물이 차오를 때는 밖으로 나가 모험을 하고, 물이 빠질 때는 기지와
광산으로 돌아와 내실을 다지는 자연스러운 플레이 흐름**을 코딩하는 것이
최종 목적입니다.

자, 이제 이 철학적 토대 위에서 마인크래프트를 가장 완벽한 1인용 RPG
패키지 게임으로 탈바꿈시킬 준비가 끝났습니다.
